# jrDev

jrDev is a practical guide designed to help students and beginner programmers become better **junior software developers**.

Many computer science programs teach programming concepts but often do not focus on the real-world development skills used in professional software teams. This project aims to bridge that gap by introducing the tools, workflows, and best practices commonly expected from junior developers.

This repository is structured as a collection of topics that developers can explore to improve their understanding of real development environments.

## Topics Covered

The project currently includes sections on:

- Git and version control
- Docker and container basics
- Software architecture fundamentals
- Debugging techniques
- Writing clean and maintainable code
- Reading and understanding existing codebases
- Testing and validation
- System design basics
- Working with APIs
- Databases
- Terminal and command line skills
- Code reviews
- Pull requests
- Development workflows
- Documentation practices
- Performance fundamentals
- Security basics
- Logging and monitoring
- Build tools
- Dependency management
- CI/CD pipelines
- Open source contributions
- Common mistakes made by junior developers
- Interview preparation

Each topic will contain short lessons and examples explaining how these concepts are used in real development environments.

## Purpose

The goal of jrDev is to help developers transition from:

**student → productive junior developer**

by learning the tools, habits, and practices that experienced developers expect on real projects.
